# デザインシステム指示書（視覚的指示書）

コードを一切見なくても、このファイルだけで AIDD ツール群と同じ見た目を再現するための指示書。CSS 変数などの**正確な数値の真実源は `skills/design-system/SKILL.md`**（このファイルでは値を再定義しない。矛盾したらスキル側が正）。

## 基準トークン（要約）

- **プライマリ**: `#1976D2`（Material Blue 700）。アクション・アクティブ状態・リンクにのみ使う
- **書体**: 本文・UI は **Noto Sans JP**、コード・数値・等幅表示は **JetBrains Mono**
- **テーマ**: **Material Design 3 のライトテーマ**が既定。ダークは明示指示がある時だけ。**ダークトークンは `skills/design-system/SKILL.md`（2026-07 追加）に定義済み**なので、必要になったら新規に配色を考えず流用する
- **ステータス色**: ISTQB severity と対応（Critical=赤 / High=橙 / Medium=黄 / Low=緑 / Info=水色）。正確な色値と背景ペア色はスキル参照

## 全パターン共通の文法

どのパターンでも守る「見た目の骨格」。

1. **地と図**: ページ背景はごく薄いグレー、コンテンツは白いカード。カードは 1px の薄いボーダー＋ごく弱い影で浮かせる（影で語らずボーダーで区切る）
2. **角丸**: 小さい部品ほど小さく（バッジ=小、カード=中、モーダル=大、ピル型は完全な丸）。四角い角の要素を作らない
3. **余白**: 4px の倍数だけを使う。カード内側は広め（24px 相当）、部品同士は 8〜16px 相当。詰まって見えたら余白を増やす方向で解決する
4. **色の禁欲**: 1画面あたり「プライマリ＋ステータス色＋グレー階調」以外を持ち込まない。装飾目的のグラデーション・彩度の高い背景は使わない
5. **タッチターゲット**: 押せるものは 44×44px 以上。片手操作の主要動作は画面下部寄り
6. **日本語 UI**: `lang="ja"`。行間はゆったり（1.7 前後）。英数字の羅列（ID・スコア・パス）は等幅フォントに切り替える

## パターン1: Web アプリ（PWA・単一 HTML ツール）

利用者が「操作して結果を得る」道具。QA ツール・家計簿 PWA などが該当。

- **構図**: デスクトップは左サイドナビ（幅240px 相当・薄いグレー地・右に 1px 境界線）＋右メインの2ペイン。ナビには機能一覧を縦に並べ、選択中の項目だけプライマリの淡い背景＋プライマリ文字色にする。モバイル（360px 幅基準）ではナビを畳み、ハンバーガーまたは下部タブに切り替える
  - **サイドバーが常時表示される SaaS 型ツールでは、サイドバーを画面最上部から独立させ、ヘッダーはメインカラム側だけに置く**（ヘッダーがサイドバーの上を横断する構造にしない）。デスクトップではサイドバー自体の折りたたみ（アイコンのみ表示）も用意する。具体的な実装は `skills/design-system/SKILL.md` の「永続サイドバー + 折りたたみ」を参照（Harness ToDo モックで検証済み）
- **階層**: ページタイトル（大・太字）→ カードタイトル（中・太字）→ 本文（14px 相当）→ メタ情報（小・グレー文字）。1画面の見出しレベルは3段まで
- **色の役割**: 主ボタンはプライマリ塗り＋白文字を**1画面に1つだけ**。それ以外のボタンは白地＋ボーダーまたはテキストボタン。判定結果・重要度は必ずステータス色のピル型バッジ（濃い文字＋同系の淡い背景）で示す
- **密度**: 一覧はカードグリッド（最小幅 300px 相当で折り返し）。1カード=1対象。カード内は「タイトル → 要約1〜2行 → バッジ列 → アクション」の縦順
- **状態**: ホバーで背景をわずかに沈める／処理中はボタン内スピナー＋文言を「実行中…」に／空状態は中央にグレーの説明文＋開始アクション／エラーは Critical 色のカラウト（左に色帯）で本文内に表示し、alert ダイアログは使わない
- **禁止**: 1画面に塗りボタン複数、ステータス色の装飾転用（例: 見出しを赤にする）、影の多用

## パターン2: HTML スライド

社内説明・デモ用のプレゼン。1ファイルで完結し、横 16:9 を基準にする。

- **構図**: 1スライド=1メッセージ。上から「キッカー（小さなプライマリ色ラベル）→ 見出し（スライド高の約1/8の文字サイズ・太字・2行以内）→ 本文領域」。左右余白はスライド幅の約8%、上下は約10%を空け、端まで文字を詰めない
- **表紙**: 白地にプライマリの細い水平線または左縦帯1本＋タイトル大書き＋日付・作成者を小さくグレーで。写真やイラストは使わない（テキストと図形のみで成立させる）
- **本文スライドの型**は3種のみ: ①箇条書き（最大5点・各1行）②比較表（ヘッダ行を薄グレー地に）③図解（角丸四角のボックスを矢印でつなぐ。ボックスは白地＋ボーダー、強調したい1つだけプライマリ淡背景）
- **階層**: 見出し > 小見出し > 本文 > 注記 の4段。本文はスクリーン投影を想定して大きめ（会議室の後ろから読めるサイズ＝1スライドに本文8行が上限の目安）
- **色の役割**: 基本はグレー階調＋プライマリ1色。数字の強調（KPI 等）だけプライマリの大きな等幅数字。良し悪しを示す時のみ Low 緑／Critical 赤を文字色に使う
- **フッター**: 全スライド共通で右下にページ番号（小・グレー・等幅）、左下に資料名。ロゴ帯や飾り罫は入れない
- **禁止**: アニメーション過多、1スライドに2メッセージ、10行を超える箇条書き、背景色付きスライド

## パターン3: 管理画面（ダッシュボード・一覧管理）

データの監視と CRUD が目的。QA ポータル・集計ダッシュボードが該当。

- **構図**: 左サイドナビは画面最上部から独立して伸ばし、折りたたみ可能にする（パターン1の補足と同じ構造）。右側だけに高さ 56px 相当の白いトップバー（左: 検索、右: 通知・アバター）＋メイン。メイン最上段に **KPI カード列**（3〜5枚横並び・各カードは「ラベル小文字グレー → 等幅の大きな数値 → 前回比を小さく」の縦順。説明が要る指標には ℹ️ アイコン＋ツールチップを添える）、その下にフィルタ行、最下段にデータテーブル
- **テーブル**: ヘッダ行は薄グレー地＋太字、罫線は横線のみ（縦線は引かない）、行ホバーで淡ハイライト。数値列は右揃え＋等幅フォント、状態列はバッジ。列見出しに列単位フィルタ（絞り込みアイコン→ポップオーバー）を持たせてよい。1画面の表示件数は可変（5/10/30/50/100/すべて）にし、フッターに現在範囲/総数とページ番号を置く（詳細は SKILL.md 参照）
- **フィルタ・検索**: テーブル直上に1行で並べる（検索ボックス→セレクト→期間→リセット）。適用中のフィルタはピル型チップで可視化し、×で個別解除できる形にする
- **階層**: 管理画面は Web アプリより情報密度を1段高くする（本文13〜14px 相当・カード内余白もやや狭く）。ただし余白の4px 倍数ルールは維持
- **色の役割**: KPI の良し悪し・行の状態はステータス色バッジで統一。グラフを置く場合も同じステータス色系列＋プライマリのみで描き、新しい色を発明しない
- **状態**: 読み込み中はテーブル領域にスケルトン行（グレーの帯）／0件は「条件に一致するデータがありません＋フィルタ解除リンク」／破壊的操作（削除）は必ず確認ダイアログ＋ Critical 色のボタン
- **禁止**: 縦罫線だらけの Excel 風テーブル、意味のない円グラフ装飾、KPI カードへの背景色塗り（白地＋数値色で語る）

## 適用除外: ドキュメント・マニュアル類

`docs/yuki-aidd-kit-manual.html` のような**読み物ページは上記3パターンの対象外**。読み物は Qiita 風の記事レイアウト（緑系アクセント・記事カード・追従サイドバー目次）を採用しており、これは意図的な別ジャンルである。共通するのは骨格（ライトテーマ・白カード＋1px ボーダー・角丸 8px・2ペイン・ピル型タグ）のみで、**アクセント色はプライマリ #1976D2 を強制しない**。読み物を作る時は manual.html を実例として参照する。

## 再現チェックリスト（他モデル向け）

作った画面を提示する前に自己確認する:

- [ ] プライマリ #1976D2 は「操作・選択・リンク」にだけ使われている（装飾に使っていない）
- [ ] 本文 Noto Sans JP／数値・コード JetBrains Mono の使い分けができている
- [ ] 塗りボタンは1画面1つ。severity 表現はピル型バッジ
- [ ] 余白がすべて 4px の倍数で、白カード＋1px ボーダー＋ライト背景の骨格になっている
- [ ] 360px 幅（キット統一のモバイル検証基準）で崩れない
- [ ] 正確な色値・サイズは `skills/design-system/SKILL.md` の変数と一致している
