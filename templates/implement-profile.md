# 実装モード プロファイル（implement-profile）

`.claude/mode` が存在する状態（実装モード）に適用する作業プロファイル。plan 確定後の実装フェーズで、再探索をやめて確定方針の実装に集中するための行動規範。`/implement` が有効化し、`block-explore.sh` hook が Read/Grep/Glob を物理ブロックする。

## 前提
- plan（`PLAN.md` 等）が確定済みであること。未確定なら実装モードに入らない（`/implement` が plan の存在をチェックし、無ければ中断する）
- 探索系ツール（Read / Grep / Glob）は実装モード中はブロックされる

## 行動規範
- **再探索しない**: 追加の Read/Grep/Glob を行わない。必要な情報は plan に記載済みという前提で進める
- **plan 準拠**: plan に列挙した対象ファイル・手順から逸脱しない
- **小さく実装 → 軽量テスト**: 1論理変更ごとにコミット。実装直後に軽量テスト、UI変更はブラウザ/スクリーンショット確認を省略しない
- **完了判定**: 「動いた」で終わらせず done-gate を通す
- **解除**: 実装完了後は `/plan` で `.claude/mode` を削除し、探索許可状態へ戻す

## 逸脱時の手順
plan に無い探索や方針変更が必要になったら:
1. 実装を止める
2. `/plan` で実装モードを解除（`.claude/mode` を削除）
3. 調査して方針を更新（PLAN.md を修正）
4. `/implement` で再び実装モードに入る
