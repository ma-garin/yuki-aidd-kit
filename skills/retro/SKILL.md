# レトロ・学びの蓄積ループ

プロジェクトの成果物ではなく「AIDDの進め方」を改善する。ここが回ると、キット自体と開発速度が継続的に上がる。

## 3層の記録（混同しない）
| ファイル | 何を記録するか |
|---|---|
| CURRENT_STATE.md | 今どこまで進んだか（進捗） |
| ADR | なぜその設計にしたか（技術判断） |
| **lessons.md** | **AIDDの進め方の知見（プロセス改善）** ← このスキルの対象 |

## トリガー
- プロジェクト完了時（必須）
- 大きくつまずいた / 手戻りが発生した時（その場で記録）
- 月1の定期レトロ（軽くてよい）

## レトロの観点（KPT変形）
- **Keep**: うまくいったAIDDのやり方（再利用したいプロンプト・フロー）
- **Problem**: 詰まった点（スキルが発火しなかった・トークンを浪費した・evalで見逃した）
- **Try**: 次に試すこと → キット（スキル/コマンド/hook）への反映候補

## lessons.md への記録形式
templates/lessons.md を使う。1エントリ3-5行。冗長にしない。

## キットへのフィードバック（重要）
lessons.mdのProblem/Tryが溜まったら、以下に反映する:
- 発火しなかったスキル → descriptionに言い回しを追加
- 繰り返した手作業 → スラッシュコマンド化
- 繰り返したミス → hookで自動検出 or done-gateに項目追加
- 有効だったプロンプト → スキル本文に定着

## 出力
レトロ実行時は、Keep/Problem/Try を整理し、lessons.mdへの追記内容と、キットへの反映候補（あれば）を提示する。
