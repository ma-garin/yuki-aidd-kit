# 単一HTMLツール開発規約

社内配布・デモ・PoC用ツールの標準形式。npm/ビルド不要、ファイル1つで完結させる。

## 絶対制約
- **単一HTMLファイル**: CSS/JSの外部分割禁止。`<style>` `<script>` をインライン記述
- 外部依存はCDN（cdnjs）のみ許可。オフライン要件がある場合はCDNも不可
- ファイルサイズ制限なし。行数が多くなったら全体再生成ではなく str_replace で部分編集
- 実装後は必ずブラウザでの動作確認手順を提示する

## デザイン規約
- **Material Design 3 / ライトテーマ**が既定（ダークテーマは明示指示がある場合のみ）
- 日本語UI。フォントは `'Noto Sans JP', 'Hiragino Sans', sans-serif`
- レイアウト: サイドナビ＋メインの2ペイン、またはカードグリッド
- 配色はCSS変数（`:root`）で一元管理。テーマ切替を後付けできる構造にする
- ステータス表示はバッジ/チップ（NEW・severity・スコア等）

## 機能パターン（過去ツールの定型）
- **データ永続化**: localStorage。キーは `{appName}:{dataType}` 形式、スキーマバージョン番号を持たせる
- **AI連携**: APIキーはlocalStorage保存＋設定モーダルで入力。ハードコード禁止
- **構造化出力**: AI応答はJSON強制（プロンプトに「JSONのみ、前置き・コードフェンス禁止」を明記し、パース時に ```json フェンス除去のフォールバックを入れる）
- **エクスポート**: Jira CSV / GitHub Issues形式 / Markdown / PDF印刷（`@media print`）
- **反復管理**: イテレーション間のdelta比較が必要なツールではスナップショットをlocalStorageに保持

## レビュー観点の組み込み
QA系ツールでは qa-review-standards スキルを併読し、ISO 25010スコアリング・ISTQB severity・Whittakerツアーをプロンプト/UIに反映する。

## バージョンアップ時
1. 既存ファイルの構造（CSS変数・関数名・data schema）を先に把握
2. localStorageスキーマ変更時はマイグレーション関数を追加
3. 変更点一覧（機能・UI・schema）を冒頭コメントに追記
